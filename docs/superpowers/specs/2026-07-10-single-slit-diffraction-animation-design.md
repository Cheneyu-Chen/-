# 单缝声衍射动态图设计说明

**目标：** 保留“进阶声学”的参数面板不变，把“单缝声衍射”右侧图像区重做成动态图，清楚展示声波入射、穿过缝口、发生绕射并在探测屏上形成条纹的过程。

**Architecture：** 现有 `AdvancedAcousticsPage` 继续作为入口，参数面板、播放控制和导出功能都保留。单缝模式改为一个状态化的动画渲染器：上半区展示传播过程，下半区展示屏幕上的强度分布随时间逐步显影。核心物理计算尽量下沉到 `app/core/advanced_acoustics.py` 的纯函数里，页面只负责把结果画出来。

**Tech Stack：** Python, PySide6, NumPy, Matplotlib

---

## 范围

- 只改 `app/pages/advanced_acoustics_page.py` 中“单缝声衍射”这一支。
- 必要时补充 `app/core/advanced_acoustics.py` 的单缝动画计算函数。
- 不改参数面板结构，不改主导航，不改其它声学模式。

## 视觉目标

### 上半区：衍射形成过程

- 显示入射波从左向右推进。
- 显示单缝前的挡板、缝口和透射后的波前。
- 波前穿过缝口后由平直转为弯曲，逐步扩散成半圆波前。
- 用颜色或透明度区分“入射波”“缝口附近”“透射波”。

### 下半区：屏幕条纹显影

- 显示探测屏的位置和最终声强包络。
- 随时间逐步显影，不把它画成真的“强度在抖动”，而是像曝光一样慢慢建立。
- 用一条移动的探针线或高亮带提示当前正在形成的区域。

## 交互规则

- 播放按钮启动动画。
- 暂停按钮停止动画。
- 重置按钮把动画时间回到起点并刷新当前画面。
- 导出按钮保存当前图像。
- 参数变化后立即重新计算并刷新，继续保持动态图效果。
- 切换到其它实验类型时，这个模式的动画状态不会影响其它页面。

## 计算模型

### 建议新增的核心函数

在 `app/core/advanced_acoustics.py` 中新增一个纯函数，用来返回单缝动画所需的数据：

```python
def single_slit_diffraction_frame(
    frequency: float,
    slit_width: float,
    screen_distance: float,
    sound_speed: float,
    time_phase: float,
    resolution: int = 240,
):
    ...
```

这个函数应输出：

- 2D 声场网格数据
- 缝口几何位置
- 屏幕上的 1D 强度曲线
- 当前帧的波前阶段信息
- 归一化后可直接绘图的数据

实现上可以用缝口离散成多个惠更斯次级波源，叠加得到透射波场。这样既直观，也和教学演示比较一致。

## 页面结构

### `AdvancedAcousticsPage`

- 保留左侧控制卡片。
- 右侧图像区改成一个固定的动态图容器。
- 单缝模式下，建议使用上下两块画布区域：
  - 上图：传播过程
  - 下图：屏幕显影
- `QTimer` 继续驱动 `time_value`，但绘图逻辑改为面向动画帧。

### 状态管理

- `time_value` 继续作为动画时间基准。
- `refresh_plot()` 按当前实验类型分发。
- `apply_preset()` 继续沿用现有接口，单缝模式下只更新频率、缝宽和屏距。
- `stop_animation()` 在页面切换时仍可被主窗口调用。

## 验证标准

- 进入“单缝声衍射”后，右侧不是静态图，而是有明显的传播过程动画。
- 上半区能看出波前穿过缝口并向外扩散。
- 下半区能看出屏幕条纹逐步形成。
- 改频率、缝宽、屏距后，动画仍能正确刷新。
- 播放、暂停、重置、导出都保持可用。
- 其它进阶声学模式的显示和行为不受影响。

## 风险与约束

- Matplotlib 每帧重绘的开销可能偏高，动画实现需要控制分辨率和刷新频率。
- 画面要避免把“显影过程”误画成真实时间变化的物理量，说明文案要清楚。
- 右侧布局改动后要确保中文文本、图例和标题都不会挤压重叠。
